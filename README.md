# DSA Class AGH

Welcome to the **Data Structures and Algorithms (DSA)** course repository at AGH University of Science and Technology! This repository contains comprehensive educational materials, code examples, and laboratory exercises for learning fundamental concepts in computer science.

## üìö About the Course

This course provides a solid foundation in data structures and algorithms, essential for every computer science student and software engineer. Through theoretical lectures and practical laboratory sessions, students will learn how to design, analyze, and implement efficient algorithms and data structures.

## üéØ Course Objectives

By completing this course, you will:

- **Understand** fundamental algorithmic paradigms and complexity analysis
- **Master** essential data structures (arrays, lists, trees, graphs, hash tables)
- **Implement** classic algorithms for sorting, searching, and graph processing
- **Analyze** time and space complexity using Big-O notation
- **Apply** dynamic programming and greedy strategies to solve optimization problems
- **Develop** problem-solving skills through hands-on coding exercises

## üìñ Table of Contents

The `course/` folder contains detailed educational materials organized by topic:

1. [**Introduction to Algorithms**](course/01_intro_algorithms.md) - Algorithm fundamentals, analysis, and design principles
2. [**Data Structures**](course/02_data_structures.md) - Stacks, queues, lists, hash tables, BST, AVL, and red-black trees
3. [**Dynamic Programming & Greedy Algorithms**](course/03_dynamic_greedy.md) - Optimization techniques and problem-solving strategies
4. [**Graph Algorithms**](course/04_graph_algorithms.md) - Representation, traversal, topological sorting, and minimum spanning trees
5. [**Pattern Matching**](course/05_pattern_matching.md) - String searching algorithms (KMP, Rabin-Karp)
6. [**Sorting Algorithms - Basics**](course/06_sorting_basics.md) - Fundamental and advanced sorting methods
7. [**Linear Time Sorting & Order Statistics**](course/07_linear_sorting.md) - Counting sort, radix sort, bucket sort, and selection algorithms

### Folder Descriptions

- **`01/` - `05/`** - Laboratory folders containing practical implementations and solutions to assignments
- **`src/`** (if present) - Source code for reusable data structures and algorithms

## üöÄ How to Use This Repository

### For Students

1. **Study the Theory**: Start with the `course/` folder materials. Read each module sequentially to build a strong foundation.
2. **Practice Coding**: After understanding the theory, work through the exercises provided in each course module.
3. **Review Lab Solutions**: Check the numbered folders (`01/`, `02/`, etc.) for practical implementations and solutions.
4. **Experiment**: Clone the repository and modify the code to deepen your understanding:
   ```bash
   git clone https://github.com/UmarlyPoeta/DSA_class.git
   cd DSA_class
   ```
5. **Compile and Run**: Most examples are in C++ or Python. Compile C++ files with:
   ```bash
   g++ -std=c++17 -o program file.cpp
   ./program
   ```

### Learning Path

1. Begin with **01_intro_algorithms.md** to understand algorithm analysis
2. Progress through **02_data_structures.md** to learn fundamental data structures
3. Study **03_dynamic_greedy.md** and **04_graph_algorithms.md** for advanced techniques
4. Complete with **05_pattern_matching.md**, **06_sorting_basics.md**, and **07_linear_sorting.md**

### For Self-Study

- Each course module includes exercises ranging from easy to difficult
- Try solving problems before looking at solutions
- Compare your implementations with those in the lab folders
- Use the references section in each module for deeper understanding

## üí° Tips for Success

> **Tip:** The `course/` folder is your personal knowledge repository. Feel free to:
> - Add your own notes and annotations
> - Implement the pseudocode in your preferred language
> - Create additional examples
> - Solve the exercises and compare with lab solutions

- **Practice regularly** - Algorithms require consistent practice
- **Understand, don't memorize** - Focus on understanding the underlying principles
- **Analyze complexity** - Always consider time and space trade-offs
- **Code it yourself** - Implement algorithms from scratch before using libraries
- **Use visualizations** - Tools like VisuAlgo can help you understand how algorithms work
